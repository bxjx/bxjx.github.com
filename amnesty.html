Which language are you most familiar with?

I'd probably have to say Javascript. It was the main language at my first job. In every job since, I've used it. I've enjoyed building increasingly more sophisticated applications with it as the languages ecosystem has evolved. Recently I used it to build a html5 web application designed to be able to be used offline and on mobile devices. I've also enjoyed using it with Node.js, where it is well adapted to high concurrency event-based architectures.

What would you say are the greatest strengths (and weaknesses) of that language as a development platform?

Weaknesses:

  1. The main weakness used to be inconsistent APIs across different browsers. This weakness has largely been overcome with abstraction libraries like Prototype.js, and in more recent times, jQuery, Modernizer and jQuery mobile.
  2. There are many parts to language that can only be avoided with knowledge of best practices: e.g. using Immediately-Invoked Function Expressions to control scope, potential for memory leaks, leaving off the var keyword etc.
  3. It's support for closures make it a great language for using functional programming, but the language is not optimsed for that usage. E.g., a lot of modern javascript makes use of anonymous functions - yet these are unweildly to use.

Strengths:

  1. Ubiquitous! Nearly any computer or mobile device can run your code
  2. Adapted for event-based architectures making it great for interface programming and high concurrency network applications.
  3. Healthy ecosystem: e.g. jquery and it's many plugins, micro frameworks like backbone.js, jasmine and rhino for automated testing.
  4. It's the ancestor of Coffescript, my current favorite language!

What's your favourite open source tool? Why?

Oooh, now this is tough! Is it Vi, is it linux itself, git, or one of the great web frameworks i've used like Drupal or Ruby on Rails? In the end, I'd have to say Firefox Mozilla. I cut my teeth in this code, so it has a special place in my heart. Mozilla always advocated free and open web technologies - which is something I believe in.

Mozilla Firefox has also had some of the best web development tools. Notable tools are Firebug and the Web Development toolbar. Firebug for long time was THE tool to use if you were doing AJAX development. Now most browsers (including Firefox) have provided native tools with similar functionality - but a lot of them are largely clones of firebug. A web developer should be familiar with all browsers, but Firefox is still probably the best to develop in.

Mozilla's commitment to open source allows others to alter and reuse it without restrictive proprietary licensing. One example of a project that has benefited from this is Tor - the p2p anonymiser project. Tor uses a modified firefox to make this technology accessible and easy to use.

I also love how the project has successfully inspired and harnessed volunteers - not unlike Amnesty!

If you were hiring for this role, what would be the one most important skill or quality you would be looking for in candidates

I would argue that the most important quality would be passion! Firstly, passion for web development. A web developer should be genuinely excited about programming and web technologies. They should program because they enjoy it. They should not have to be reminded to explore new technologies and methodologies - a passionate programmer never stops learning. I would consider hiring an experienced passionate programmer even if they did not currently possess all the desired skills. In time, a good passionate programmer will learn the skills better than a less motivated programmer that already has the experience. Passion also often indicates mastery - or at least, a desire to master a skill.

Secondly, they should be passionate about Amnesty and the causes it supports. A candidate should believe that this work is important and worth doing.
